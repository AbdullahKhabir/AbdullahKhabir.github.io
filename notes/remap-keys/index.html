# Keyboard remapping

There are many programs out there for X. But want to do it better way with systemd.

# Get keyboard info

 
cat /proc/bus/input/devices

I: Bus=0011 Vendor=0001 Product=0001 Version=ab54
N: Name="AT Translated Set 2 keyboard"
P: Phys=isa0060/serio0/input0
S: Sysfs=/devices/platform/i8042/serio0/input/input0
U: Uniq=
H: Handlers=sysrq kbd leds event0
B: PROP=0
B: EV=120013
B: KEY=402000000 3803078f800d001 fefff79fffefffff fffffffffffffffe
B: MSC=10
B: LED=7

And check the keyboard info… You will need handlers line. In that line search for event number. For example event0.

Copy the line.

# Install evtest

 
nix-env -i evtest

Now hit the key you want to remap.

# Get key info

Run evtest on the event you found in first step. event0.

evtest /dev/input/event0

# event0 is from last step.

# Here the output goes...

Event: time 1659355203.806256, -------------- SYN_REPORT ------------
Event: time 1659355204.310583, type 4 (EV_MSC), code 4 (MSC_SCAN), value db
Event: time 1659355204.310583, type 1 (EV_KEY), code 125 (KEY_LEFTMETA), value 0

And search for key value. It will be there in first line and last value after (MSC_SCAN). I have modified it so don’t follow blindly. Instead press the key you want to remap and search for the value. In above example it is db.

Write down it.

# Get desired key info

Next step is to find the keycode of the key you want this new key remapped to.

It is located in /usr/include/linux/input-event-codes.h. NixOS is an exception so just use find to find this file.

#define KEY_LEFTMETA		125
#define KEY_RIGHTMETA		126

Copy the keycode. In this example, it’s LEFTMETA.

# Create configuration file

Now create a configuration file.

And paste the information you copied earlier from first step.

The first line should be:

evdev:input:b<bus_id>v<vendor_id>p<product_id>e<version_id>-<modalias>
  • evdev: Leave it as it is.
  • input: Input devices identifed by USB kernel modalias. For built in keyboard and AT keyboards, put atkbd. If you have some USB keyboard, put keyboard:usb:v*p* for whole line.

All other blocks you can copy from the first step. The first line…

I: Bus=0011 Vendor=0001 Product=0001 Version=ab54


vim /etc/udev/hwdb.d/my-customkeys.hwdb


evdev:input:b0011v0001p0001*
 KEYBOARD_KEY_cf=leftmeta
 KEYBOARD_KEY_c7=esc

Don’t forget to put a space in the beginning of the lines where you remap the keys. In this example second and third line… Also you will find the desired keys keycodes in UPPERCASE in 4th step, just use them here in lowercase else things won’t work…

# Update hardware database

 

systemd-hwdb update

# Reload database

 

udevadm trigger

Enjoy…
